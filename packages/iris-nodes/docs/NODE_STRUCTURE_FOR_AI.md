# Iris 워크플로우 노드 개념 설명서 (for AI)

> 이 문서는 **다른 AI 에이전트**가 Iris 워크플로우의 노드 시스템을 개념적으로
> 이해하도록 작성되었습니다. 코드 구조·타입·빌드 절차는 다루지 않고, **어떤
> 노드가 있으며 각각 무엇을 하는지**만 설명합니다.

---

## 1. Iris 워크플로우란

Iris는 **노드 기반 비주얼 프로그래밍**으로 AI 자동화를 만드는 도구입니다.
사용자는 캔버스에 "노드(블록)"를 배치하고, 노드의 **출력 포트**와 다른 노드의
**입력 포트**를 선으로 이어 데이터가 흐르는 파이프라인을 구성합니다.

기본 정신 모형:

```
[트리거] → [생성/분석/편집/가공] → [더 가공] → [출력]
   진입점        중간 처리 단계                    결과 전송
```

- **노드**: 하나의 작업 단위. (예: "이미지 생성", "HTTP 요청", "조건 분기")
- **포트**: 노드의 입출력 연결점. 각 포트에는 데이터 타입이 있음.
- **엣지(연결선)**: 한 노드의 출력 포트 → 다른 노드의 입력 포트로 데이터 전달.
- **설정(config)**: 노드별 동작을 조정하는 폼 값 (연결선이 아니라 사용자가 직접 입력).

데이터에는 8가지 타입이 있고, 포트끼리 연결할 때 호환성 판단에 쓰입니다:
**text(텍스트) · image(이미지) · video(영상) · audio(음성) · document(문서) ·
json(구조화 객체) · any(아무 타입) · trigger(실행 신호)**.

`trigger` 타입은 데이터가 아니라 **"이제 실행해"라는 신호**입니다. 모든 진입점
노드가 이 신호를 내보내며, 이걸로 워크플로우 실행 순서가 결정됩니다.

노드는 **7개 카테고리**로 나뉩니다. 색상으로도 구분됩니다:

| 카테고리 | 색 | 역할 | 개수 |
|---------|-----|------|------|
| TRIGGER | 초록 | 워크플로우를 시작시키는 진입점 | 8 |
| GENERATOR | 보라 | AI로 새 콘텐츠를 생성 | 14 |
| ANALYZER | 파랑 | 미디어/문서/텍스트를 분석·추출·분류 | 8 |
| EDITOR | 주황 | 기존 미디어를 편집·변환 | 21 |
| UTILITY | 회색 | 흐름 제어 + 데이터 가공 | 23 |
| WEB | 남색 | 웹에서 데이터 수집 | 3 |
| OUTPUT | 청록 | 결과를 외부로 전송·저장 | 6 |

## 2. TRIGGER — 진입점 (8개)

워크플로우가 "언제, 무엇으로" 시작되는지를 정의합니다. 입력 포트가 없고
`trigger` 신호 + 초기 데이터를 내보냅니다.

- **Manual Trigger** — 사용자가 수동으로 실행. 텍스트/이미지/파일을 직접 입력해
  넣을 수 있음. (가장 기본적인 진입점)
- **Schedule Trigger** — 정해진 주기(cron)로 자동 실행. 고정 입력값을 함께 전달.
  이전 실행이 아직 안 끝났을 때의 동작(건너뛰기/대기/병렬)도 설정.
- **Webhook Trigger** — 외부에서 HTTP POST가 오면 실행. 요청 본문(payload)과
  헤더를 그대로 내보냄.
- **Event Trigger** — 시스템 이벤트(파일 업로드 완료, 워크플로우 완료, 커스텀
  이벤트 등)에 반응해 실행.
- **Directory Watch** — (데스크톱 전용) 특정 폴더에 새 파일이 생기면 실행.
- **Chat Trigger** — 채팅 UI에서 사용자가 메시지를 보내면 실행. 대화 기록을
  유지하므로 멀티턴 챗봇의 표준 진입점. (에이전트 모드와 결합하면 사내 봇 구축)
- **Form Trigger** — 커스텀 폼을 노출하고, 사용자가 제출하면 실행. 비기술
  사용자가 안전한 입력 검증과 함께 워크플로우를 돌리게 해줌.
- **Email Received** — 새 이메일이 도착하면 실행(Gmail Push 또는 IMAP polling).
  발신자/제목 필터 지원. 인박스 자동화(티켓 분류, 자동 응답)의 진입점.

## 3. GENERATOR — AI 콘텐츠 생성 (14개)

프롬프트나 입력 미디어로부터 **새로운 콘텐츠를 만들어냅니다.** 대부분
provider/model 선택이 필요합니다.

- **Text Generator** — LLM으로 텍스트 생성. **두 가지 모드**:
  - *Single Call*: 일반 한 번의 LLM 호출 (기본값).
  - *Agent*: LLM이 다른 노드를 **도구로 반복 호출**하며 스스로 판단해 작업을
    완료하는 ReAct 루프 (반복 횟수 상한 설정). 이 노드가 Iris에서 "에이전트"를
    구현하는 핵심.
- **Image Generator** — 텍스트 → 이미지. 종횡비, 카메라 앵글, 주체(subject)
  일관성, 업스케일/배경제거 후처리 옵션.
- **Image to Image** — 입력 이미지 + 프롬프트로 변형. 변형 강도 조절.
- **Text to Video** — 텍스트 → 영상.
- **Image to Video** — 이미지를 움직이는 영상으로(첫/끝 프레임 지원).
- **Text to Speech** — 텍스트 → 음성(음성 종류, 속도 선택).
- **Speech to Text** — 음성 → 전사 텍스트(언어 선택/자동 감지).
- **Generate Subtitles** — 영상에서 AI 전사로 SRT/VTT 자막 생성.
- **Text to Music** — 텍스트/가사 → 음악(Suno). 스타일·보컬 성별·인스트루멘탈 옵션.
- **Inpaint** — 마스크로 가린 영역을 AI 생성 콘텐츠로 채움.
- **Outpaint** — 이미지를 원래 경계 너머로 확장.
- **Style Transfer** — 한 이미지의 화풍을 다른 이미지에 입힘.
- **Face Swap** — 두 이미지 간 얼굴 교체.
- **Lip Sync** — 영상 속 인물의 입모양을 별도 오디오에 동기화(AI 더빙/아바타).

> GENERATOR vs EDITOR 구분: GENERATOR는 "없던 콘텐츠를 만든다"는 관점,
> EDITOR는 "있는 미디어를 손본다"는 관점. (Inpaint처럼 양쪽에 비슷한 게 있음)

## 4. ANALYZER — 분석·추출·분류 (8개)

미디어나 텍스트를 **이해해서 정보를 끄집어냅니다.** 새 미디어를 만들지 않고,
텍스트/JSON 같은 "지식"을 출력합니다.

- **Image Analyzer** — 이미지를 설명·객체 탐지·OCR·커스텀 질의.
- **Video Analyzer** — 영상 내용 분석(프레임 샘플링 주기 설정).
- **Text Analyzer** — 텍스트 감정 분석·개체 추출·요약·키워드 추출.
- **Audio Analyzer** — 음성을 전사한 뒤 GPT로 분석(요약/화자 식별 등).
- **Document Analyzer** — PDF 페이지·스캔 문서·이미지를 비전 모델로 분석.
- **Long-Context Document Q&A** — 파일을 통째로 long-context 모델에 넣어 질의응답.
  **RAG(벡터 DB)의 대안** — 청킹/임베딩 없이 prompt cache로 비용 절감. 같은 파일
  반복 질의 시 ~90% 절감.
- **Structured Extract** — 입력에서 **지정한 JSON Schema 모양으로** 구조화 데이터
  추출(이름/이메일 등). 자유 텍스트 생성과 달리 정해진 스키마를 보장.
- **Categorize** — 입력을 미리 정의한 카테고리 중 하나(또는 여럿)로 분류.
  다중 라벨·"미분류 허용" 옵션.

---

## 5. EDITOR — 미디어 편집·변환 (21개)

이미 존재하는 이미지/영상/음성을 **변형·후처리**합니다.

**이미지 편집**
- **Image Upscale** — 해상도 향상(2x/4x).
- **Image Inpaint** — 마스크 영역을 채우거나 수정.
- **Image Outpaint** — 경계 너머로 확장.
- **Style Transfer** — 화풍/스타일 적용(강도 조절).
- **Face Swap** — 얼굴 교체.
- **Background Remove** — 배경 제거(마스크도 함께 출력).
- **Sky Replace** — (데스크톱) 하늘을 AI 생성 하늘로 교체.
- **Relight** — (데스크톱) 조명 방향·분위기 재조정.
- **Auto Enhance** — (데스크톱) 원클릭 선명도·색·디테일 보정.
- **Image Crop** — 지정 비율로 자르기.
- **Image Filter** — 색 필터(흑백/세피아 등) + 밝기/대비/채도 조정.
- **Mask Define** — 인페인팅용 마스크 영역 정의(AI 자동/수동).

**영상 편집**
- **Motion Control** — 참조 영상의 움직임을 참조 이미지 속 인물에 전이.
- **Video Upscale** — AI 해상도/프레임레이트 향상.
- **Video Inpaint** — 마스크로 영상 내 객체 제거/채움.
- **Video Trim** — 시간 구간 잘라내기.
- **Video Crop** — 화면 비율로 자르기.
- **Video Merge** — 여러 클립을 시간 순서로 이어 붙임(트랜지션 옵션). AI가 만든
  짧은 클립들을 긴 영상으로 합칠 때 필수.
- **Video Overlay** — 영상 위에 텍스트/이미지/워터마크 합성(위치/투명도/시간 구간).

**음성 편집**
- **Audio Trim** — 시간 구간 잘라내기.
- **Audio Separate** — 음성을 보컬/반주/드럼/베이스 stem으로 분리(리믹스/노래방).

## 6. UTILITY — 흐름 제어 + 데이터 가공 (23개)

AI가 아닌 **프로그래밍 로직**을 담당합니다. 분기·반복·변환·계산 등 워크플로우의
"접착제"이자 "배관". 여러 노드는 에이전트의 도구로도 쓰입니다.

**흐름 제어**
- **Delay** — 지정 시간만큼 대기.
- **Condition** — 단순 조건(같음/포함/크다 등)으로 true/false 분기.
- **Conditional** — (데스크톱) JS 표현식으로 분기.
- **Router** — N방향 분기. 여러 조건 중 **처음 맞는 가지**로 보내고, 없으면
  default로. (n8n/Zapier의 멀티 분기에 해당)
- **Filter** — 조건을 만족하면 통과(passed), 아니면 버림(rejected). 루프 안에서
  원치 않는 항목을 거를 때 유용.
- **Loop** — 배열을 순회하며 항목을 하나씩 내보냄. 끝나면 done 신호.
- **Aggregate** — 루프 각 반복의 결과를 **하나로 모음**(배열/객체/문자열). Loop와
  짝을 이룸.
- **Try / Catch** — 상류 작업의 에러를 잡아 error 출력으로 우회(워크플로우 전체가
  죽지 않게). 지수 백오프 재시도 지원.
- **Sub-Workflow** — 다른 저장된 워크플로우를 **한 스텝으로 실행**. 재사용 가능한
  서브루틴 모듈화. 동기/비동기 실행 선택.

**데이터 가공**
- **Merge** — 여러 입력을 객체/배열/문자열로 합침.
- **Split** — 입력을 구분자로 둘로 쪼갬.
- **Transform** — 대소문자/공백 정리/**JSON 파싱**/문자열화 변환. (특히 문자열을
  객체로 바꾸는 `parseJson`이 자주 필요 — 9절 참조)
- **Template** — `{{변수}}` 자리표시자가 있는 텍스트를 데이터로 채워 렌더. 결과는
  **문자열**.
- **HTTP Request** — 임의의 REST API 호출(GET/POST 등, 헤더/쿼리/경로 파라미터/
  본문). 외부 서비스 연동의 만능 도구.
- **Script** — 커스텀 JavaScript 실행(`input` 변수로 받아 결과 return).
- **Regex** — 정규식 매칭/추출/치환.
- **Date** — 날짜 파싱/포맷/덧셈/차이 계산(타임존 지원).
- **JSON Path** — JSONPath나 점·괄호 표기로 JSON에서 값 추출.
- **Document Grep** — 파일 안에서 키워드/정규식 검색 + 주변 줄 반환. Long-Context
  Q&A가 비쌀 때의 가벼운 대안(grep+read 패턴).

**상태/파일**
- **Set Variable / Get Variable** — 이름 붙인 변수에 값을 저장/조회(노드 간 상태 공유).
- **File Save / File Load** — (데스크톱) 파일 경로에 데이터 저장/로드.

---

## 7. WEB — 웹 데이터 수집 (3개)

외부 웹에서 정보를 가져옵니다. 모두 에이전트의 도구로 쓸 수 있습니다.

- **Web Search** — Perplexity Sonar로 웹 검색(제목/URL/스니펫 반환). 동일 질의는
  60분간 캐시되어 추가 비용 없음.
- **Web Scraper** — URL을 정제된 마크다운으로 변환(Firecrawl/Jina/Readability).
  광고·네비 제거, 본문 추출. SPA 렌더링 대기 옵션.
- **YouTube Transcript** — 유튜브 URL → 자막 텍스트(공식 자막 우선, 없으면
  Whisper로 음성 인식). 타임스탬프·세그먼트 옵션.

---

## 8. OUTPUT — 결과 전송·저장 (6개)

워크플로우 결과를 **외부 세계로 내보냅니다.** 보통 워크플로우의 종착점.

- **Save to Storage** — 클라우드 스토리지에 저장하고 URL 반환.
- **Webhook Output** — 결과를 외부 HTTP 엔드포인트로 전송.
- **Send Email** — 이메일로 결과 발송(첨부 지원).
- **Notification** — 푸시/Slack/Discord 알림 발송.
- **Slack Post** — Slack 채널/DM에 메시지 발행(Block Kit 리치 메시지, 스레드 답글).
- **Sheet Append** — Google Sheets에 행 추가(컬럼 매핑). 결과를 CRM/리포트/로그로 기록.

## 9. AI가 알아야 할 동작 규칙

노드를 조합해 워크플로우를 만들 때, 노드 설명만으로는 드러나지 않는 런타임
계약들입니다. **이걸 모르면 "그럴듯하지만 실행되면 깨지는" 워크플로우가 나옵니다.**

1. **포트 타입 호환성** — 출력 포트의 타입과 입력 포트의 타입이 맞아야 연결됨
   (`any`는 아무거나 허용). 영상 출력을 텍스트 입력에 꽂을 수 없음.

2. **문자열 vs 객체 — 가장 흔한 함정**:
   - 많은 노드가 **문자열**을 내보냅니다: Manual Trigger의 text(사용자가 JSON을
     입력해도 원시 문자열), Template의 result, Text Generator의 text 등.
   - 반면 **HTTP Request의 본문**이나 구조화 API는 **객체**를 기대합니다.
   - 문자열을 객체가 필요한 곳에 바로 넣으면 이중 인코딩되어 API가 400을 반환.
   - **해결**: 사이에 `Transform`(parseJson)을 끼워 문자열→객체로 변환.
   - 이미 객체인 출력(Webhook payload, 다른 HTTP 응답, Transform 출력)은 그대로 OK.

3. **에이전트와 도구** — Text Generator를 Agent 모드로 두면 다른 노드를 도구로
   호출합니다. 단, **도구로 노출 가능한 노드는 부작용이 가벼운 것뿐**입니다
   (Web Search, HTTP Request, Script, Structured Extract, Long-Context Q&A,
   Regex/Date/JSON Path, Document Grep, Sub-Workflow 등). 출력 노드, 비가역 편집,
   돈을 쓰는 미디어 생성은 에이전트가 함부로 못 쓰게 도구에서 제외됨.

4. **provider/model** — AI 호출 노드는 provider와 model을 골라야 합니다. 각 노드의
   "AI 능력"(예: text-to-image, speech-to-text)에 맞는 provider만 후보로 노출됨.

5. **루프 ↔ 집계 짝** — Loop로 항목을 하나씩 처리한 뒤 결과를 합치려면 Aggregate를
   하류에 두어야 합니다. Aggregate는 Loop의 done 신호가 올 때 한 번 내보냄.

6. **URL 템플릿팅** — HTTP Request나 Webhook Output의 URL은 `{{토큰}}` 자리표시자를
   경로/쿼리 입력으로 채울 수 있습니다.

7. **단일 소스 관리** — 모든 노드 정의는 한 곳(iris-nodes)에만 존재하며, 웹/데스크톱
   에디터·서버 실행 엔진·LLM 워크플로우 생성 프롬프트·SDK가 전부 이 정의를
   공유합니다. 그래서 어느 플랫폼에서 보든 노드의 포트·설정·동작은 동일합니다.
